TOP 01 — 削減効果 最大

画像補正依頼アプリ
（Python GUI）

Desktop Application / COM Integration
年間 360時間
1件10分短縮 × 年間2,000件超

顧客対応チーム（9名）・制作チーム（2名）が日常業務として利用。

PythonPySide6Google Sheets APIChat WebhookPyInstallerCOM連携

改善前

  • サムネイルを目視で確認
  • ファイル名を手動変更
  • フォルダを手動で移動
  • 依頼フォームを手動入力
  • チャットで担当者に連絡

改善後

  • HEIC→JPG 自動変換
  • ファイル名 自動リネーム
  • フォルダ 自動移動
  • スプレッドシート 自動記録
  • Google Chat 自動通知
  • Adobe Photoshop 自動起動（COM連携）
技術・実装の詳細を見る
課題の背景

CS担当9名が毎日こなす写真補正依頼が、サムネイル目視・ファイルリネーム・フォルダ移動・フォーム入力・チャット連絡という5工程で構成されていた。1件10分 × 年間2,000件超 ＝ 年間330時間以上。格納漏れ・連絡漏れも常態化。

技術選定の理由
  • 社内PC（Windows）で動く実行ファイルが必要 → Python + PyInstaller で .exe 化
  • GUI必須（非エンジニアが使う）→ PySide6 でドラッグ＆ドロップUI
  • 既存の Photoshop を自動制御 → Windows COM連携（win32com）
  • 進捗共有は既存のスプレッドシートを活用 → Sheets API で同期
実装の工夫
  • iPhone撮影のHEIC画像をJPGへ自動変換（pillow-heif）
  • 案件グループNoから法人名・医院名を自動補完
  • 右クリックでトリミング範囲を描画指示できるUI
  • 30分バックグラウンド監視 → Photoshop自動起動・ファイル展開
  • 補正完了後、所定フォルダへの自動移動 + Chat通知まで一気通貫
開発規模 / 運用状況

設計〜実装〜配布まで単独担当。CS9名 + 制作2名の計11名が日常業務として利用中（2024年〜）。ヒューマンエラー発生率 ほぼ0。

取材手配の全自動化（GAS）

Workflow Automation / Google Workspace
年間 226時間
1件20分 → 3分 / 年間800件対応

案件IDによる全情報の一元管理。4部署が同一フローを利用。

GASGmail APICalendar APIDrive APIChat API

改善前

  • メールを手動作成・送信
  • カレンダーへ手動登録
  • Driveフォルダを手動作成

  • カレンダー 自動登録
  • 5営業日前 自動リマインド
  • 日程変更・キャンセル 自動連動
  • 30分ごと納品 自動巡回 → Chat通知
技術・実装の詳細を見る
課題の背景

求人誌・パンフ制作での取材確定後、カレンダー登録・ドライブ作成・メール下書き3通（社内・クライアント・ライター宛）・チャット共有を毎回20分かけて手動対応。転記ミス・リマインド漏れが常態化し、業務が完全に属人化していた。

技術選定の理由
  • 既存の Google Workspace（Sheet・Gmail・Calendar・Drive・Chat）を活用 → 追加コストゼロ
  • GASはサーバー不要・ブラウザからデプロイ可能 → 運用コストゼロ
  • トリガー（onEdit）で入力時に自動発火 → バッチ遅延なし
実装の工夫
  • 案件IDで全情報を一元管理 → 4部署が同一フローで運用
  • 土日祝を除外して「5営業日前」を自動逆算するカレンダーロジック
  • ステータス変更を検知してカレンダー・ドライブを自動削除するキャンセル連動
  • 30分ごとにDriveをクロールして納品検知 → リンクボタン付きChat通知
  • 宛先・内容が異なる3種のメールテンプレートを自動生成
開発規模 / 運用状況

設計〜実装〜運用移行まで単独担当。4部署・年間800件の運用を担う基幹フローとして現在も稼働中。誤送信・ダブルブッキングをゼロ化。

TOP 03 — 定常業務の置換

週次レポート自動生成
（GA4 + Looker Studio）

Data Pipeline / Visualization
年間 150時間
毎週3時間 → 15分未満

データ取得から配信までを完全自動化。毎朝、関係者へ自動で届く。

GA4BigQueryLooker StudioGAS

改善前

  • GA4から手動でデータ取得
  • スプレッドシートへ手動転記
  • グラフを手動更新・共有

改善後

  • BigQueryへ 自動同期
  • Looker Studioで 自動可視化
  • 毎朝メール 自動配信
技術・実装の詳細を見る
課題の背景

毎週月曜朝、GA4から手動でデータを取得 → スプレッドシートへ転記 → グラフ更新 → 関係者へメール送信という作業に3時間を消費。担当者固定の完全属人業務で、休暇時には報告が止まっていた。

技術選定の理由
  • GA4 → BigQuery へのエクスポートで構造化データを取得
  • Looker Studio でリアルタイム可視化（毎週更新不要）
  • GAS のScheduled Trigger で毎朝7:00に自動配信
  • 既存 Google Workspace 内で完結 → 追加コストゼロ
実装の工夫
  • BigQuery のテーブル設計を週次集計に最適化
  • Looker Studio レポートURLをメール本文に埋め込み → 受信者がワンクリックで最新データへアクセス
  • 担当者不在でも確実に届くスケジューラ設計
開発規模 / 運用状況

設計〜実装〜定常運用まで単独担当。週3時間の定常作業を15分未満（確認のみ）に圧縮。関係者への週次配信は2023年以降フル自動で継続中。
